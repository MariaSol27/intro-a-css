Glosario de Términos de GIT

GIT
Un sistema de control de versiones distribuido utilizado para rastrear cambios en el código fuente a lo largo del tiempo.
Repositorio
Un repositorio contiene todos los archivos de un proyecto (incluyendo la documentación), y almacena el histórico de modificaciones de cada archivo.
Commit
Un commit se puede entender como la confirmación de una modificación individual en un archivo (o serie de archivos).
Branch
Una línea de desarrollo independiente en un repositorio. Permite trabajar en características o correcciones sin afectar la rama principal.
Merge
El proceso de combinar cambios de una rama a otra. Usualmente se hace desde una rama de características a la rama principal.
Pull
Comando utilizado para descargar y fusionar cambios del repositorio remoto a la copia local del repositorio.
Push
Se refiere a enviar tus cambios confirmados (tus commits) a un repositorio remoto.
Clone
Método para crear una copia local del repositorio remoto, con todo su historial de commits.
Fork
Una copia del repositorio en tu propia cuenta de GitHub. Permite realizar cambios sin afectar el repositorio original.
